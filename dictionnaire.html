Dictionnaire de Données

Qu'est-ce qu'un dictionnaire de données ?

Un dictionnaire de données est un document qui décrit les éléments de données utilisés dans un système d'information ou une base de données. Il permet de comprendre la structure et les relations entre les différentes données dans un projet.

Pourquoi faire un dictionnaire de données ?

  • Compréhension des données : Il aide toutes les personnes impliquées dans le projet à comprendre comment les données sont structurées.
  • Documentation : Il sert de documentation claire et précise, ce qui facilite la gestion des données sur le long terme.
  • Normalisation et cohérence : Il garantit que les données sont définies de manière uniforme et que les règles de gestion des données sont respectées.
  • Facilitation de la maintenance : Il permet de comprendre rapidement la structure des données, surtout lors des mises à jour.
  • Intégrité des données : Il aide à maintenir les relations entre les entités, comme les clés étrangères, et à garantir que les données sont valides.

Dictionnaire de Données

Table : Produit

Nom du champ Type Contraintes Description
id INT PRIMARY KEY, NOT NULL Identifiant unique du produit
libelleCourt VARCHAR(50) NOT NULL Libellé court du produit
libelleLong VARCHAR(255) Description longue du produit
referenceFournisseur VARCHAR(50) Référence fournisseur du produit
prixAchat DECIMAL(10, 2) NOT NULL, CHECK(prixAchat >= 0) Prix d'achat du produit
prixVente DECIMAL(10, 2) CHECK(prixVente >= 0) Prix de vente calculé
stock INT CHECK(stock >= 0) Quantité en stock
actif BOOLEAN NOT NULL Indique si le produit est actif
photo VARCHAR(255) Lien ou chemin de l'image du produit
categorie_id INT FOREIGN KEY (Categorie), NOT NULL Référence vers la catégorie
fournisseur_id INT FOREIGN KEY (Fournisseur), NOT NULL Référence vers le fournisseur

Explication des choix :

  • id (INT, PRIMARY KEY, NOT NULL) : L'identifiant unique du produit, utilisé pour différencier chaque enregistrement dans la base de données.
  • libelleCourt (VARCHAR(50), NOT NULL) : Ce champ est limité à 50 caractères pour éviter des noms trop longs et inefficaces en termes de stockage.
  • libelleLong (VARCHAR(255)) : Permet de stocker des descriptions plus longues du produit. Le choix de 255 caractères est standard pour des chaînes de texte longues.
  • referenceFournisseur (VARCHAR(50)) : Stocke la référence du fournisseur, un champ souvent court mais critique pour identifier les produits externes.
  • prixAchat (DECIMAL(10, 2), NOT NULL, CHECK(prixAchat >= 0)) : `DECIMAL(10, 2)` assure une précision sur les montants financiers. La contrainte `CHECK` empêche l'insertion de valeurs négatives pour ce champ.
  • prixVente (DECIMAL(10, 2), CHECK(prixVente >= 0)) : Même logique que pour `prixAchat`, ce champ stocke le prix de vente du produit avec deux décimales et la contrainte `CHECK` empêche les valeurs négatives.
  • stock (INT, CHECK(stock >= 0)) : Représente la quantité disponible du produit en entier. La contrainte `CHECK` garantit que le stock ne soit jamais négatif.
  • actif (BOOLEAN, NOT NULL) : Ce champ binaire indique si le produit est actif (disponible) ou non.
  • photo (VARCHAR(255)) : Stocke le chemin ou l'URL de l'image du produit, généralement un champ texte long, d'où l'utilisation de 255 caractères.
  • categorie_id (INT, FOREIGN KEY, NOT NULL) : Référence la catégorie du produit via une clé étrangère. Chaque produit doit être associé à une catégorie valide.
  • fournisseur_id (INT, FOREIGN KEY, NOT NULL) : De même, chaque produit doit avoir un fournisseur, et cette relation est définie par une clé étrangère.
